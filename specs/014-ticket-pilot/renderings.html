SpecKit Pilot — Autonomous Ticket → PR

A revamp of the existing SpecKit Pilot extension. The spec-kit lifecycle stays as the orchestration backbone; we add a ticket front-door that seeds Specify, an autonomous agent runner that executes each phase, and two new tail phases — Self-Review and Open PR.
Extension: terminator.speckit-pilot Surface: project tab (unchanged) Runtime: Claude Agent SDK runner Phases: 8 spec-kit + 2 new
Spec Kit is the orchestration layer. The SDD cycle it already runs — /speckit-specify → clarify → plan → checklist → tasks → analyze → implement, with its artifacts and review gates — is the engine that drives every run. This product wraps that engine in the new flow and layout below: a ticket front-door that seeds specify, an agent runner that drives each spec-kit command, human gates between phases, and two tail phases (Self-Review, Open PR). The old SpecKit Pilot UI is fully replaced. Green = new capability.
