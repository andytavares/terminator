## Entities

### PhaseId

```typescript
type PhaseId =

  | 'plan'
  | 'checklist'
  | 'tasks'
  | 'analyze'
```

Ordered — phase index determines upstream/downstream relationships.

---

### PhaseStatus

```typescript
type PhaseStatus =
  | 'locked' // upstream not approved
  | 'ready' // upstream approved, not yet run
  | 'running' // command injected, awaiting artifact
  | 'awaiting_review' // artifact written, waiting for gate decision
  | 'approved' // user approved this phase's artifact
  | 'stale' // approved but upstream hash changed since approval
  | 'modified' // approved artifact edited by user; re-approval required
  | 'failed' // run errored (timeout, no artifact produced)
```

**Valid transitions:**

```
locked        → ready          (upstream approved)
ready         → running        (user triggers run)
running       → awaiting_review (artifact detected via fs:changed)
running       → failed          (timeout: no artifact within configuredTimeoutMs)
failed        → ready           (user retries or edits prompt)
awaiting_review → approved     (user approves)
awaiting_review → ready        (user rejects)
approved      → stale          (upstream artifact hash changed)
approved      → modified       (artifact file changed on disk)
modified      → awaiting_review (user saves edits or re-runs)
stale         → running        (user re-runs)
approved      → ready          (user revokes approval)
```

### PhaseState

```typescript
interface PhaseState {
  id: PhaseId
  status: PhaseStatus
  approvedHash: string | null // SHA-256 of artifact at time of approval
  approvedAt: string | null // ISO 8601 timestamp
  approvedBy: string | null // "Name <email>" from git config
  lastRunId: string | null // UUID of most recent run
  lastRunAt: string | null // ISO 8601 timestamp
  artifactPaths: string[] // relative to repo root
}
```

```typescript
interface PilotState {
```

### PilotSettings

```typescript
interface PilotSettings {
  defaultModel: string // e.g. "claude-opus-4-6"
  phaseGates: Record<PhaseId, PhaseGateConfig>
  disallowedPaths: string[] // glob patterns, e.g. [".env*", "secrets/**"]
  maxFilesPerImplementRun: number // default: 25
  maxTokensPerCommand: number // default: 50000
  commandTimeoutMs: number // default: 300000 (5 min)
  requireCleanTreeForImplement: boolean // default: true
  createCheckpointBeforeImplement: boolean // default: true
  runConsolePosition: 'bottom' | 'side' | 'tab'
  reviewerIdentity: 'git' | 'os' | 'custom'
  customReviewerName: string | null
  branchConvention: 'sequential' | 'feature-slash' | 'custom'
  customBranchPattern: string | null
  openSidebarOnStart: boolean
}

interface PhaseGateConfig {
  required: boolean // false = phase can run without upstream approval (for 'checklist')
  autoApprove: boolean // default: false
  perFileConfirm: boolean // only applicable for 'implement'
}
```

### HistoryEntry (one line in `.specify/.pilot/history.jsonl`)

```typescript
interface HistoryEntry {
  ts: string // ISO 8601
  actor: string // "Name <email>"
  action:
    | 'run_start'
    | 'run_complete'
    | 'run_failed'
    | 'approved'
    | 'rejected'
    | 'revoked'
    | 'modified'
    | 'stale' // auto-generated by system
    | 'file_approved' // implement per-file
    | 'file_skipped' // implement per-file
  phase: PhaseId
  runId?: string // UUID, present for run_* actions
  hash?: string // artifact hash at event time
  inputs?: string[] // artifact paths used as inputs
  note?: string // user-provided note
  filePath?: string // for file_approved / file_skipped actions
}
```

---

### RunRecord (in-memory only, not persisted to state.json)

```typescript
interface RunRecord {
  id: string // UUID
  phase: PhaseId
  startedAt: string // ISO 8601
  sessionId: string // Terminator terminal session ID
  commandInjected: string // the full slash command string sent
  status: 'running' | 'completed' | 'failed' | 'stopped'
  pendingFileWrites: PendingFileWrite[] // implement phase only
}

interface PendingFileWrite {
  taskId: string // e.g. "T-04"
  filePath: string // relative to repo root
  isNew: boolean
  linesAdded: number
  linesRemoved: number
  diffContent: string // unified diff string
  decision: 'pending' | 'approved' | 'skipped'
}
```

```typescript
interface Feature {
```

## State Persistence Boundaries

| Entity                               | Stored? | Where                                         |
| ------------------------------------ | ------- | --------------------------------------------- |
| PilotState (phase states + settings) | Yes     | `.specify/.pilot/state.json`                  |
| HistoryEntry                         | Yes     | `.specify/.pilot/history.jsonl` (append-only) |
| RunRecord                            | No      | In-memory only (Zustand store)                |
| Feature list                         | No      | Derived from specs/ directory scan            |
| Active session                       | No      | Derived from api.terminal.onSessionCreate     |

## Artifact Paths per Phase

| Phase        | Output Artifact(s)                                                                              |
| ------------ | ----------------------------------------------------------------------------------------------- |
| constitution | `.specify/memory/constitution.md`                                                               |
| specify      | `specs/<feature>/spec.md`                                                                       |
| clarify      | `specs/<feature>/spec.md` (Clarifications section)                                              |
| plan         | `specs/<feature>/plan.md`, `research.md`, `data-model.md`, `contracts/*`                        |
| checklist    | `specs/<feature>/checklists/requirements.md`                                                    |
| tasks        | `specs/<feature>/tasks.md`                                                                      |
| analyze      | `specs/<feature>/analyze.md` _(or stdout — treated as no output file; auto-approves after run)_ |
| implement    | Any files written under repo root                                                               |
