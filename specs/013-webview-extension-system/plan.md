# Implementation Plan: Webview-Isolated Extension System

**Branch**: `eextension` | **Date**: 2026-06-24 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `specs/013-webview-extension-system/spec.md`

## Summary

Replace the broken dual-context extension renderer (Vite `import.meta.glob` build-time coupling + dynamic ESM import into shared JS context) with a `WebContentsView`-isolated extension system modeled on VSCode/Cursor. Every extension UI runs in a separate Electron `WebContentsView` with its own renderer process, loaded from a manifest-declared HTML entry. The core app never runs extension renderer code. Any extension can be installed from any filesystem path without rebuilding the core app.

## Technical Context

**Language/Version**: TypeScript 5.x / Node.js 22 (Electron 42.4.1)
**Primary Dependencies**: Electron 42.4.1, React 18.3.1, electron-vite, Zustand, Zod
**Storage**: `electron-store` (userData directory JSON files) for extension registry; extension assets served via `ext://` custom protocol
**Testing**: Vitest (unit + component tests), `@testing-library/react`
**Target Platform**: macOS / Windows / Linux desktop (Electron)
**Project Type**: Desktop application вЂ” extension subsystem overhaul
**Performance Goals**: All extension `WebContentsView` instances created and loading by the time the app window is shown (< 3s from launch); extension UI visible within 3s of first tab activation; full reload within 5s of clicking "Reload" in Settings
**Constraints**: Extension main entries must be CJS (Node `require()` in extension-host); extensions cannot import from core source paths; `WebContentsView` bounds must be updated with `devicePixelRatio` applied
**Scale/Scope**: 4 extensions migrated (notepad, git-integration, task-vault, speckit-pilot); 1 new SDK package; ~13 core files modified; litmus test: any extension installable from any directory

## Implementation Order

### Phase 1 вЂ” Core Infrastructure

Complete in order. Each step has a paired test. Run `npm run lint && npx vitest run --coverage` after step 12.

1. **`electron.vite.config.ts`** вЂ” Add `webview: resolve(__dirname, 'src/main/preload-webview.ts')` to preload input. No test needed (build config).

2. **`src/main/preload-webview.ts`** (new file) вЂ” Identical API surface to `src/main/preload.ts`. Exposes `window.electronAPI` via `contextBridge`. Compiled to `dist-electron/preload/webview.js`. Test: verify it exports the same surface shape as preload.ts (import+type-check only).

3. **`src/shared/schemas/extension.schema.ts`** вЂ” Add `ExtensionContributesSchema` (globalTab, workspaceTab, projectTab, sidebarPanel, windowViews, commands). Add `contributes` field to `ExtensionManifestSchema`. Export `ExtensionContributes` type. Write spec: schema validation unit tests.

4. **`src/shared/types/index.ts`** вЂ” Add `ExtensionContributes` interface. Add `contributes?: ExtensionContributes` to `Extension`. Covered by schema spec from step 3.

5. **`src/main/extensions/extension-view-host.ts`** (new file) вЂ” `ExtensionViewHost` class:

   - `constructor(mainWindow: BrowserWindow)`
   - `createView(ext: Extension): void` вЂ” creates `WebContentsView` with preload, loads `ext://id/...` URL, adds to `mainWindow.contentView`, wires `did-finish-load` в†’ `extension:panel-loaded` push
   - `destroyView(id: string): void`
   - `reloadView(id: string): void` вЂ” calls `view.webContents.reload()`
   - `handleBoundsUpdate(id: string, bounds, visible, dpr): void` вЂ” applies `setBounds()` + `setVisible()`
   - `broadcastToAll(channel: string, data: unknown): void` вЂ” sends to all view `webContents`
   - Write spec: `extension-view-host.spec.ts` with Electron mocks.

6. **`src/main/extensions/extension-host.ts`** вЂ” Parse `contributes` from manifest in `load()`. Store on `ExtensionRecord`. Return in `listExtensions()`. Update existing spec to cover new field.

7. **`src/main/index.ts`** вЂ” Remove `webviewTag: true` (was never added; confirm it's absent). Init `ExtensionViewHost` after window creation. Wire `extension:update-panel-bounds` ipcMain handler в†’ `viewHost.handleBoundsUpdate()`. Pass `viewHost.broadcastToAll` as broadcast to `registerExtensionHandlers`. Add `workspace:active-changed` ipcMain handler в†’ `viewHost.broadcastToAll('workspace:changed', data)`.

8. **`src/main/ipc/extension.ipc.ts`** вЂ” Accept `broadcast: (channel: string, data: unknown) => void` param. After successful `extension:reload`, call `broadcast('extension:renderer-reload', { id })` and call `viewHost.reloadView(id)`.

9. **`src/main/preload.ts`** вЂ” Add:

   - `extensionEvents.onExtensionPanelLoaded(handler: (id: string) => void): () => void` вЂ” listens for `extension:panel-loaded`
   - `extensionEvents.onExtensionRendererReload(handler: (id: string) => void): () => void` вЂ” listens for `extension:renderer-reload`
     Update existing preload spec.

10. **`src/renderer/electron.d.ts`** вЂ” Add matching type signatures for the two new `extensionEvents` methods. No test needed (type-only).

11. **`src/renderer/components/ExtensionPanelPortal.tsx`** (new file):

    ```tsx
    interface Props {
      extensionId: string
      extensionName: string
      viewParam?: string
      isActive: boolean
    }
    ```

    - `useRef<HTMLDivElement>` for the portal container
    - `useState(true)` for `loading`
    - `useEffect` with `ResizeObserver` в†’ sends `extension:update-panel-bounds` IPC on every resize and on `isActive` change
    - `useEffect` listening for `onExtensionPanelLoaded(id)` в†’ `setLoading(false)` when `id === extensionId`
    - `useEffect` listening for `onExtensionRendererReload(id)` в†’ re-sends bounds (view was recreated)
    - Renders: container div (full width/height) + conditional loading spinner overlay
      Write spec: `ExtensionPanelPortal.spec.tsx`.

12. **`src/renderer/extensions/loader.ts`** вЂ” Rewrite `initExtensions()`:
    - Remove `dynamicLoader` import and usage
    - Remove `window.__terminatorRegistry` global
    - For extensions with `contributes` (new model): call `registerWebviewExtension(ext)` which calls the appropriate `registry.*` methods, each receiving `<ExtensionPanelPortal extensionId={id} viewParam={view} isActive={...} />`
    - Keep `import.meta.glob` path for un-migrated bundled extensions (detect: no `contributes` field)
      Update loader spec.
