# Data Model: Markdown Notepad Extension

**Feature**: `specs/010-markdown-notepad`  
**Date**: 2026-06-18  
**Storage**: SQLite (WAL mode) — `userData/notepad.db`

---

## Entity Overview

```
notes ─── note_tags ─── tags
  │
  └── comments (self-ref: parent_id, max depth 2)

notes ──(rowid)── notes_fts   (FTS5 virtual table, contentless)
notes ──(key)──── settings
```

---

## Tables

### `notes`

System of record for all note content.

| Column        | Type | Constraints                      | Description                                             |
| ------------- | ---- | -------------------------------- | ------------------------------------------------------- |
| `id`          | TEXT | PRIMARY KEY                      | UUID v4, generated by Node `crypto.randomUUID()`        |
| `title`       | TEXT | NOT NULL DEFAULT 'Untitled note' | Display title; derived from body if empty on save       |
| `body`        | TEXT | NOT NULL DEFAULT ''              | Raw markdown source                                     |
| `created_at`  | TEXT | NOT NULL                         | ISO 8601 UTC timestamp                                  |
| `updated_at`  | TEXT | NOT NULL                         | ISO 8601 UTC timestamp; updated on every autosave       |
| `archived_at` | TEXT | NULL                             | Set when note is archived (soft-delete). NULL = active. |

**State transitions**:

- Active: `archived_at IS NULL`
- Archived: `archived_at IS NOT NULL` — hidden from note list and search by default; visible in Archived section and with "Include archived" toggle
- Permanently deleted: row removed (hard-delete, requires confirmation)

**Indexes**: Primary key on `id`. No additional indexes needed at v1 scale.

---

### `tags`

Canonical tag registry. Tags are first-class rows — renaming one row renames it everywhere.

| Column | Type | Constraints     | Description                                                                                                        |
| ------ | ---- | --------------- | ------------------------------------------------------------------------------------------------------------------ |
| `id`   | TEXT | PRIMARY KEY     | UUID v4                                                                                                            |
| `name` | TEXT | UNIQUE NOT NULL | Lowercase tag name (e.g. `infra`, `idea`). Inline `#tag` tokens are normalized to lowercase before reconciliation. |

---

### `note_tags`

Many-to-many join between notes and tags.

| Column    | Type | Constraints                                     | Description                      |
| --------- | ---- | ----------------------------------------------- | -------------------------------- |
| `note_id` | TEXT | NOT NULL REFERENCES notes(id) ON DELETE CASCADE |                                  |
| `tag_id`  | TEXT | NOT NULL REFERENCES tags(id) ON DELETE CASCADE  |                                  |
|           |      | PRIMARY KEY (note_id, tag_id)                   | Composite PK prevents duplicates |

**Cascade behavior**: Deleting a note cascades to `note_tags`. Deleting a tag cascades to `note_tags` only — the note is not deleted.

---

### `comments`

Text annotations anchored to a range in a note. Flat threading: max depth 2 (top-level + replies; replies cannot be replied to).

| Column         | Type    | Constraints                                     | Description                                                                                         |
| -------------- | ------- | ----------------------------------------------- | --------------------------------------------------------------------------------------------------- |
| `id`           | TEXT    | PRIMARY KEY                                     | UUID v4                                                                                             |
| `note_id`      | TEXT    | NOT NULL REFERENCES notes(id) ON DELETE CASCADE | Parent note                                                                                         |
| `parent_id`    | TEXT    | NULL REFERENCES comments(id) ON DELETE CASCADE  | NULL for top-level; set for replies. Replies do not accept further replies (enforced in IPC layer). |
| `body`         | TEXT    | NOT NULL                                        | Comment text (plain text; no markdown rendering in v1)                                              |
| `author`       | TEXT    | NOT NULL DEFAULT 'me'                           | Single-user; always 'me' in v1                                                                      |
| `status`       | TEXT    | NOT NULL DEFAULT 'open'                         | `'open'` \| `'resolved'` \| `'orphaned'`                                                            |
| `start_offset` | INTEGER | NULL                                            | Byte offset of anchor start in note body. NULL for replies.                                         |
| `end_offset`   | INTEGER | NULL                                            | Byte offset of anchor end in note body. NULL for replies.                                           |
| `quote`        | TEXT    | NULL                                            | Exact anchored text snapshot. NULL for replies.                                                     |
| `prefix`       | TEXT    | NULL                                            | Up to 32 chars before the anchor (for disambiguation). NULL for replies.                            |
| `suffix`       | TEXT    | NULL                                            | Up to 32 chars after the anchor (for disambiguation). NULL for replies.                             |
| `created_at`   | TEXT    | NOT NULL                                        | ISO 8601 UTC                                                                                        |
| `updated_at`   | TEXT    | NOT NULL                                        | ISO 8601 UTC                                                                                        |

**Anchor data rules**:

- Top-level comments (`parent_id IS NULL`): MUST have `start_offset`, `end_offset`, `quote`. `prefix`/`suffix` SHOULD be populated (may be empty string at document boundaries).
- Reply comments (`parent_id IS NOT NULL`): anchor columns MUST be NULL. Replies display under the parent card in the margin.

**Status transitions**:

```
open → resolved    (user clicks "Resolve")
resolved → open    (user clicks "Unresolve")
open → orphaned    (re-anchor fails on note load)
orphaned → open    (manual restore if body is later recovered — not in v1, deferred)
```

**Index**: `CREATE INDEX idx_comments_note ON comments(note_id, status)`

### `notes_fts` (virtual)

Contentless FTS5 virtual table for ranked full-text search. Maintained in the same transaction as note writes.

```sql
CREATE VIRTUAL TABLE notes_fts USING fts5(
  title,
  body,
  tags,            -- space-joined tag names, refreshed on every note save
  content='',      -- contentless: we own all inserts/deletes
  tokenize='unicode61'
);
```

**Maintenance protocol**:

- Note insert → `INSERT OR REPLACE INTO notes_fts(rowid, title, body, tags) VALUES (notes.rowid, ?, ?, ?)`
- Note update → same `INSERT OR REPLACE` (replaces by rowid)
- Note delete/archive → `INSERT INTO notes_fts(notes_fts, rowid) VALUES('delete', ?)`

**Search query pattern**:

```sql
SELECT n.id, n.title, n.updated_at,
       snippet(notes_fts, 1, '<mark>', '</mark>', '…', 32) AS snippet
  FROM notes_fts
  JOIN notes n ON n.rowid = notes_fts.rowid
 WHERE notes_fts MATCH ?
   AND n.archived_at IS NULL          -- excluded when "Include archived" is off
 ORDER BY bm25(notes_fts)
```

## Schema SQL

```sql
CREATE TABLE IF NOT EXISTS notes (
  id          TEXT PRIMARY KEY,
  title       TEXT NOT NULL DEFAULT 'Untitled note',
  body        TEXT NOT NULL DEFAULT '',
  created_at  TEXT NOT NULL,
  updated_at  TEXT NOT NULL,
  archived_at TEXT
);

CREATE TABLE IF NOT EXISTS tags (
  id   TEXT PRIMARY KEY,
  name TEXT UNIQUE NOT NULL
);

CREATE TABLE IF NOT EXISTS note_tags (
  note_id TEXT NOT NULL REFERENCES notes(id) ON DELETE CASCADE,
  tag_id  TEXT NOT NULL REFERENCES tags(id)  ON DELETE CASCADE,
  PRIMARY KEY (note_id, tag_id)
);

CREATE TABLE IF NOT EXISTS comments (
  id           TEXT PRIMARY KEY,
  note_id      TEXT NOT NULL REFERENCES notes(id) ON DELETE CASCADE,
  parent_id    TEXT REFERENCES comments(id) ON DELETE CASCADE,
  body         TEXT NOT NULL,
  author       TEXT NOT NULL DEFAULT 'me',
  status       TEXT NOT NULL DEFAULT 'open',
  start_offset INTEGER,
  end_offset   INTEGER,
  quote        TEXT,
  prefix       TEXT,
  suffix       TEXT,
  created_at   TEXT NOT NULL,
  updated_at   TEXT NOT NULL
);
CREATE INDEX IF NOT EXISTS idx_comments_note ON comments(note_id, status);

CREATE VIRTUAL TABLE IF NOT EXISTS notes_fts USING fts5(
  title, body, tags,
  content='',
  tokenize='unicode61'
);

CREATE TABLE IF NOT EXISTS settings (
  key   TEXT PRIMARY KEY,
  value TEXT NOT NULL
);
```

## Domain Types (TypeScript)

```typescript
interface Note {
  id: string
  title: string
  body: string
  createdAt: string // ISO 8601
  updatedAt: string
  archivedAt: string | null
  tags: string[] // tag names, joined from note_tags
}

interface Tag {
  id: string
  name: string
  noteCount: number // computed on list queries
}

interface Comment {
  id: string
  noteId: string
  parentId: string | null
  body: string
  author: string
  status: 'open' | 'resolved' | 'orphaned'
  // Anchor (null for replies)
  startOffset: number | null
  endOffset: number | null
  quote: string | null
  prefix: string | null
  suffix: string | null
  createdAt: string
  updatedAt: string
  replies: Comment[] // populated by IPC layer for top-level comments
}

interface SearchResult {
  id: string
  title: string
  snippet: string // HTML with <mark> highlights from snippet()
  tags: string[]
  updatedAt: string
}

interface ExportFrontmatter {
  id: string
  title: string
  tags: string[]
  created: string
  updated: string
}
```

## Migration Strategy

Following Task Vault's `hasColumn` idempotent runner — all schema changes use `ALTER TABLE ... ADD COLUMN IF NOT EXISTS` or `CREATE TABLE IF NOT EXISTS`. Safe to re-run on every app launch.

v1 ships with the base schema above (no migrations needed on first run). Future migrations will use the `hasColumn(db, table, column)` helper already present in the codebase.
